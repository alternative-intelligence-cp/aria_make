Git-Aware File Exclusion in GlobEngine

PRIORITY: MEDIUM
STATUS: Partially Covered (manual exclusions work)
SOURCE: gem4_07.txt, Verification Chunk 5

PROBLEM:
GlobEngine traverses entire project tree including node_modules/, .cache/, build artifacts. Performance degrades from O(N_source) to O(N_total) in large repos. Duplicate exclusion configuration (.gitignore + aria.json).

REQUIREMENTS:
1. Automatic .gitignore parsing:
   - Read .gitignore from project root
   - Convert shell globs to std::regex
   - Augment existing exclusion set

2. Subtree pruning optimization:
   - Use disable_recursion_pending() on excluded directories
   - Reduce I/O from 100,000+ files to ~500 source files

3. Pattern support:
   - Standard globs: *.o, build/
   - Recursive wildcards: logs/**/debug.log
   - Rooted patterns: /TODO (only in root)
   - Directory-only: dist/ (directories, not files named "dist")
